# 로깅 개선 2

### 팀논의

팀 내에서 로깅시에 직접적으로 RDB에 적재하는 방향으로 검토 

- ES 부하 등 우려사항으로 전달해준 내용을 축소하기 위함
- 단, 크리티컬한 장애로그 발생시 [trace.id](http://trace.id) 로 ES에 검색할 수 있게 하는 방향(?)
    - 장애 로깅 같은 경우 전체적인 [trace.id](http://trace.id) 로 로깅을 조회하여 판단할 수 있어야함.
    - 장애 로깅이 쌓이게 되면 직접 키바나에 들어가 검색할 수 있음. 하지만 시각화에는 한계가 존재.

### **현재 모니터링 알림 방식**

```java
try {
	도메인 로직...
} catch (Exception e) {
	mail.send("에러가 발생하였습니다 - 탈퇴자", "회원 탈퇴 실패"); 
	or
	log.send("[Critical]");
}
```

- 현 방식은 에러 메일함에 컨벤션 없이 모든 알림, 모니터링, 장애 메일을 한번에 받아보고 있음.
- 메일 컨벤션을 잘 맞춘다고 하여도 개인별로 메일함 필터링 등등이 필요함.
